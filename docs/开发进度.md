- 拆分剧情、状态、结局、存档、剧情图和 UI 运行层。
- 建立严格类型检查、生产构建、自动测试与旧存档迁移。

## 已完成：0.3.0 正式序章

- 增加故事模式与县中模式。
- 将错题本升级为六格可操作版面，并建立承诺账本。
- 把序章结局、模式、版面、承诺和长期状态转换为第一章开局档案。

## 已完成：0.5.0 第二章纵向原型

- 建立四周、每周十四格的可编辑时间表与承诺占用。
- 实现周棠主动改约、保护同学而缺席和定性周复盘。
- 完成座位传页、句子拼装、纸页处理与四阶段一模。
- 将座位路线、同学关系、沟通习惯和纸页处理回写到长期状态。
- 接入第二章成绩单解释、异步留言、错峰公交和第三章复印件钩子。
- 升级 v4 存档，支持三个手动槽、自动存档和旧版本迁移。

## 已完成：0.6.0 公开试玩版

状态：已发布。

- 统一标题、版本、章节目录与第二章真实可玩状态。
- 将三条异步留言收敛到实际十八字预算，并加入完整性测试。

- 增加 Open Graph 分享信息、在线试玩文档和 GitHub Pages 自动部署。

- 当前自动测试覆盖剧情图、五种序章结局、状态、存档迁移、四周排程、座位、句子、一模、第二章状态链、章节解锁和通关总结。

## 已完成：0.7.0 核心循环重制

- 第一章每周从“至少安排三格”改为六次主动安排，承诺占用不再挤占玩家决策次数。
- 增加三项目标面板，实时说明学业、关系、恢复与边界目标是否达成。
- 四周各增加三个计划执行事件，共十二个新场景与三十六种现场行动。
- 排程内容会改变事件叙述；现场行动继续回写状态、学业方法、行为倾向、人物压力和长期事实。
- 执行事件完成后再进入座位传页、周复盘、句子拼装或一模，形成“计划—执行—核心交互—反馈”循环。
- 旧存档可继续恢复；新增执行阶段支持自动存档和三个手动档位。

## 当前里程碑：0.8.0 可玩性强化

- 每周执行事件后加入三轮压力挑战，重复使用同一规则并按周改变突发局面。
- 建立任务积压、被注意、关系负荷三条会继续推进的压力轨道。
- 学习、恢复、同学关系和协商类排程会转化为有限次数的挑战行动。
- 玩家始终可以选择硬顶，但代价会进入精力、压力、睡眠债和逆反。
- 挑战结果分为局面受控、勉强接住和压力失控，并回写周复盘与后续状态。
- 挑战中途与结算状态均可存档恢复。
