# 《晚自习之后》技术架构

## 技术选择

正式版采用 Vite、TypeScript 与原生 DOM。叙事游戏不需要通用组件框架；保留原生界面可以减少运行时负担，同时通过 TypeScript、模块化和自动测试获得长期维护能力。

生产构建使用相对资源路径，可以被静态服务器、普通网站或后续桌面封装共同承载。

## 约束

- 剧情数据不直接操作 DOM。
- 数值变化、结局判定和存档迁移必须是可单独测试的纯逻辑。
- 新节点必须通过剧情图检查，不允许断链、意外死路或不可达内容。
- 第一章活动、句段和考试动作必须通过内容注册表检查；开发时可运行 `npm run check:content`。
- 存档必须显式携带版本号；正式版字段变化通过迁移处理，不能直接废弃试玩存档。
- 正式版核心系统优先以第一章纵向切片验证，验证后再泛化到八章。
- 日程、小游戏与考试是与线性对白播放器并列的可恢复位置，不伪装成剧情 overlay。
- 所有结算都是纯状态转换后一次性保存；重新渲染和读档不得重复发放效果。
- 承诺执行依据稳定 ID 注册表，不解析 `cadence` 展示文案。
- 周棠行动按固定规则确定，不用随机数，也不由亲密值取消她的隐私边界。

## 自动化验证

`npm test` 当前覆盖：

- 剧情节点和选项规模基线。
- 全节点可达、无断链、无意外死路。
- 穷举真实选项后五种试玩结局均可达。
- 数值上下限、隐藏共担负值与派生逆反。
- 结局优先级。
- 旧存档迁移及损坏存档清理。
- 错题本版面切换、效果与稳定决策标识。
- 第一章开局档案与保留未约定状态。
- 每周十四个稳定格位、承诺锁格、纯排程编辑与四周推进。
- 三种序章承诺均可完成第一章；过量见面由周棠确定性提出改约。
- 座位路线、句子组合、隐私边界与三种纸页状态。
- 一模成绩只由长期准备与当场方法生成，不提供直接分数选项。
- 第二章成绩单、异步留言和公交路线均为纯状态转换，并可从 v4 非剧情位置恢复。
- v1/v2/v3 到 v4 迁移，以及非剧情位置的存读档。
