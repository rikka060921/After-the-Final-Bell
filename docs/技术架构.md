# 《晚自习之后》技术架构

## 技术选择

正式版采用 Vite、TypeScript 与原生 DOM。叙事游戏不需要通用组件框架；保留原生界面可以减少运行时负担，同时通过 TypeScript、模块化和自动测试获得长期维护能力。

生产构建使用相对资源路径，可以被静态服务器、普通网站或后续桌面封装共同承载。

## 模块边界

```text
index.html
  └─ src/main.ts          页面事件、剧情播放和 UI 协调
       ├─ config.ts       版本、素材路径、初始状态和显示元数据
       ├─ story.ts        纯剧情节点与选择数据
       ├─ endings.ts      结局文本和纯结局判定
       ├─ state.ts        纯数值变化与派生逆反逻辑
       ├─ save.ts         存档创建、校验和版本迁移
       ├─ story-graph.ts  断链、死路与不可达节点检查
       └─ types.ts        模块之间共享的数据契约
```

## 约束

- 剧情数据不直接操作 DOM。
- 数值变化、结局判定和存档迁移必须是可单独测试的纯逻辑。
- 新节点必须通过剧情图检查，不允许断链、意外死路或不可达内容。
- 存档必须显式携带版本号；正式版字段变化通过迁移处理，不能直接废弃试玩存档。
- 正式版核心系统优先以第一章纵向切片验证，验证后再泛化到八章。
